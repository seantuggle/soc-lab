🔇 Suppression Rules
Active suppressions prevent new alerts from being created by the detection engine.
How suppressions work: Suppressions are checked by the detection engine before an alert is written. If a rule fires and an active suppression matches (by global, host, or user scope), no alert is created — but the rule hit is still logged with a [SUPPRESSED] tag. Suppressions are created from individual alert investigation pages. They can be temporary (1h / 24h / 7d) or permanent.
